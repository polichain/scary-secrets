Scary Secrets Vault Documentation

Scary Secrets Vault is a decentralized web application created for the Halloween Hackathon, where users can share spooky stories and Halloween-themed files in a secure and decentralized way. The project leverages Storacha’s decentralized storage technologies and IPFS to manage files and Content Identifiers (CIDs), providing a fun and secure environment to explore Halloween content.

Project Overview

Scary Secrets Vault allows users to upload, explore, and view content related to Halloween, like spooky stories, images, or other files. The platform integrates with Storacha Network to ensure that files are stored in secure, decentralized spaces, while also allowing public access through IPFS. Users can interact with the platform via a user-friendly interface, and the platform takes care of the complexities of decentralized technologies.

User Navigation

The user flow is straightforward, offering a seamless experience for interacting with Halloween-themed content. Below is a typical user journey:

1. Login

Users start by logging in with their email addresses. The login system sends a verification email, and after confirmation, the user is authenticated.

2. Uploading Spooky Content

Once logged in, users can upload files, such as stories, images, or other Halloween-related content. The system automatically creates a space for the user to store their files securely. Each file is assigned a Content Identifier (CID) via IPFS, making the content publicly accessible if shared.

3. Exploring Uploaded Stories

Users can explore scary stories or other files by providing the CID of the desired content. The platform fetches the content from IPFS and displays it to the user. Users can also explore stories shared by others by entering the corresponding CID.

4. Sharing Content

After uploading a file, users receive a CID that can be shared with others, allowing public access to their content. Anyone with the CID can view the content through the platform.

Purpose of the Project

The Scary Secrets Vault was designed with a Halloween theme to encourage the sharing of spooky, creative content while using cutting-edge Web3 technologies. The platform provides a secure, decentralized solution for sharing content and introduces users to decentralized technologies, such as Storacha and IPFS, in a simple and engaging way.

Technologies Used

Frontend

  • React: Used for building the user interface. React’s component-based structure makes it scalable and easy to maintain.
  • TypeScript: Provides static typing to enhance code quality, reducing runtime errors and improving the development process.
  • CSS: Used for styling the application with a spooky, Halloween-themed look and feel.

Web3 & Decentralized Storage

  • Storacha Network: The core decentralized storage solution used to manage the storage of files in secure, permissioned DID spaces. It ensures privacy and data sovereignty.
  • IPFS (InterPlanetary File System): Files are accessible through CIDs, which allow public sharing. The application uses IPFS to ensure that content is decentralized and accessible globally.
  • Web3-Storage/W3UP Client: A library used to interface with Storacha and IPFS, enabling uploading, managing DID spaces, and creating delegations for other users.

How Technologies Were Used

The Scary Secrets Vault integrates the following technologies from Storacha:

  • DID Spaces: Each user’s files are stored in a unique, decentralized DID space, providing secure storage for the content they upload. This ensures privacy and protection for all users' data.
  • Content Identifiers (CIDs): For each file uploaded, a CID is generated using IPFS. This CID acts as a unique identifier for the file, which users can share with others to provide access.
  • Decentralized File Management: Through the use of Storacha’s Web3 client libraries, the application manages file uploads, space creation, and CID generation without needing centralized infrastructure.

Project Setup and Running Locally

Prerequisites

  • Node.js and npm/yarn installed on your machine.
  • A Storacha Network account for accessing decentralized storage.

Steps to Run Locally


# Clone the repository
git clone https://github.com/polichain/scary-secrets.git
cd scary-secrets

# Install dependencies
yarn install

# Start the development server
yarn start

# Visit the application at http://localhost:3000
